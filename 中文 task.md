# Take-Home 作业：服务电话录音分析

## 概述
在本作业中，你将分析一段涉及现场技术人员与客户的服务电话语音录音。目标是产出一份全面的通话分析，用于评估技术人员的表现并识别潜在的销售机会。你需要创建一个小型 Web 应用，以易于阅读的形式展示通话分析结果。本作业预计在约一天内完成，重点在于有洞察力的分析，而不是复杂的软件架构。

## 目标

- **通话转写（Transcribe the Call）：** 将提供的音频录音转换为文本。  
  - *提示：* 你可以使用任何工具或 API 进行转写。例如，AssemblyAI 提供 **50 美元免费额度**，可覆盖约 **185 小时**的音频 [1]。

- **合规性检查（Compliance Check）：** 判断技术人员是否遵循标准的服务通话流程。重点检查关键通话阶段是否出现、以及质量如何：
  - **开场介绍（Introduction）：** 技术人员是否恰当地向客户问候，并介绍自己/公司？
  - **问题诊断（Problem Diagnosis）：** 技术人员如何询问并理解客户的问题？
  - **解决方案说明（Solution Explanation）：** 技术人员是否清晰解释了采取的解决方案或执行的服务内容？
  - **加售/增购尝试（Upsell Attempts）：** 记录技术人员是否、以及如何尝试推销额外服务或产品。
  - **维护计划/长期服务协议（Maintenance Plan Offer）：** 技术人员是否提供任何维护计划或长期服务协议？
  - **结束与致谢（Closing & Thank You）：** 技术人员如何结束通话？是否感谢客户并礼貌收尾？

- **通话类型识别（Call Type Identification）：** 根据对话内容识别该服务电话的类型（例如：维修、例行维护、安装等）。

- **销售洞察（Sales Insights）：** 标注通话中任何销售信号或机会。例如：客户是否对额外服务表现出兴趣，或技术人员是否错过了加售线索？请给出对销售机会方面“做得好/遗漏了什么”的洞察。

## 要求

- **可访问的线上 Web 应用（Hosted Web Application）：** 开发一个简单的 Web 应用来展示你的分析。应用需要在线部署（例如使用免费托管平台或你自己的服务器），使我们可以通过 URL 直接访问。

- **转写与分析展示（Transcript & Analysis Display）：** 应用需同时展示通话转写文本与分析内容，便于评审者阅读转写并查看你对每个部分的注释或评论。
  - 你可以按合规性检查中列出的阶段（开场、诊断等）对转写进行分段，并在每段下方提供你的点评。
  - UI/UX 需强调清晰与可读性。重点是让评审者能快速理解你的洞察，而不是炫酷的设计。

- **分析深度（Depth of Analysis）：** 我们更关注分析质量，而非复杂的代码库。你不需要构建重型后端或从零实现复杂算法（鼓励使用现成 AI 工具进行转写或初步分析）。但你的评论与洞察应当详细、周到。

- **工具使用（Use of Tools）：** 你可以使用任何工具或库（包括用于转写甚至初步分析的 AI 服务）。但最终的结论必须体现你对通话的真实理解。如果使用 AI 协助分析，请进行交叉验证并确保洞察准确。

- **自包含方案（Self-Contained Solution）：** Web 应用需由你自行托管，并确保我们无需任何特殊配置即可访问。  
  - 例如，你可以部署到 **GitHub Pages、Vercel、Heroku** 等平台，或任何你选择的托管方式。

## 交付物（Deliverables）

1. **已部署 Web 应用的 URL：** 提供一个可访问的线上链接，用于查看转写与分析结果。
2. **源代码（可选）：** 虽然我们不会进行深入代码审查，但请提供源代码仓库链接或 zip 文件。主要用于验证真实性并了解你的方案组织方式（同样，我们不会深度评估架构）。
3. **说明（如有）：** 若应用需要任何特定凭证或运行步骤（对托管应用不应需要），请提供清晰说明。

## 评估标准（Evaluation Criteria）

- **分析洞察力（Insightfulness of Analysis）：** 你对通话理解得有多深入？是否捕捉到各阶段的重要细节？我们会检查你是否识别了关键合规要素（介绍、诊断等）以及任何销售机会或遗漏信号。

- **呈现清晰度（Clarity of Presentation）：** 分析应组织良好、易于跟随。使用标题、要点列表或时间戳，将你的评论与转写对齐。我们应能快速浏览并掌握关键点。

- **完整性（Completeness）：** 确保覆盖所有要求：转写、合规步骤、通话类型与销售洞察。能够逐项全面回答的分析会更突出。

- **投入与质量（Effort and Quality）：** 尽管这是一天范围的作业，我们仍期望交付足够精致：应用干净可用、文字经校对、并严格遵循说明。

- **资源使用（Use of Resources）：** 鼓励聪明地使用可用资源（如转写 API 或 AI）。我们不考核你从零实现转写引擎，而更看重你如何利用资源把精力集中在分析上。请注明或提及你使用的主要第三方工具。

## 补充说明（Additional Notes）

- **时间管理（Time Management）：** 本作业范围设计为一天可完成。请聚焦核心任务（转写与分析）。应用保持简单没问题——只要能直接有效地表达你的发现即可。

- **数据隐私（Data Privacy）：** 如果通话录音包含任何敏感信息，请保密处理。（本作业假设你有权限将数据用于分析，但不要在项目之外分享。）

- **疑问（Questions）：** 如有任何不清楚之处，欢迎联系以获取澄清。
